تصحيح تحليل خاطئ شائع

ادعاء: "missing return قبل exceptions = bug فاتالة"
غلط تماماً
كثير من المراجعات ادّعت أن الكود التالي خطأ لأنه لا يوجد return قبل الـ exception functions. هذا الادعاء غلط بالكامل، لأن الـ functions دي جوّاها throw new Error(...)، والـ throw يوقف التنفيذ تلقائياً بغض النظر عن وجود return أو لا.
src/utils/ApiError.js : line 6
// ApiError.js — الـ errorResponce دايماً بتعمل throw export const errorResponce = (...) => { throw new Error(message, { cause: { status, extra } }); // ← يوقف التنفيذ هنا }; // auth.service.js — الكود ده صح تماماً if (!user) { NotFoundException({ message: "User not found" }); // بيعمل throw → يوقف } // السطر ده لن يُنفَّذ أبداً لو user = null const isMatch = await comparePassword(password, user.password);

كيف يشتغل الـ flow بشكل صح:

NotFoundException()
→
throw new Error()
→
async fn rejects
→
.catch(next)
→
globalErrorHandler → 404
